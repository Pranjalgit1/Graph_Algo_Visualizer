# Graph Algorithm Visualizer - Documentation

This document contains brief explanations of the core classes and functions in the project.

## `algorithms` Package

### `BFS.java`
- `run(Graph graph, int start)`: Executes Breadth-First Search on the graph starting from the given vertex. Returns a list of chronological steps representing the algorithm's execution for the animator.

### `DFS.java`
- `run(Graph graph, int start)`: Executes Depth-First Search on the graph starting from the given vertex. Returns a list of execution steps.
- `dfsRecurse(...)`: Recursive helper that performs the actual DFS traversal and records each step.

### `Kruskal.java`
- `run(Graph graph)`: Executes Kruskal's Minimum Spanning Tree algorithm on the graph and returns a list of layout steps.

### `DisjointSet.java`
- `DisjointSet(int n)`: Initializes the union-find disjoint set with `n` elements.
- `find(int x)`: Finds the root representative of the set containing element `x` (uses path compression).
- `union(int x, int y)`: Merges the sets containing elements `x` and `y` (uses union by rank).
- `connected(int x, int y)`: Checks whether two elements belong to the same set.

---

## `graph` Package

### `Graph.java`
- `Graph(boolean directed)`: Creates a new adjacency-list based graph.
- `addVertex(int v)`: Adds a vertex to the graph.
- `addEdge(int u, int v)`: Adds an unweighted edge.
- `addWeightedEdge(int u, int v, int w)`: Adds a weighted edge between `u` and `v`.
- `getNeighbors(int v)`: Returns a list of `[neighbor, weight]` pairs for a vertex.
- `getVertices()`, `getEdgeCount()`, `hasVertex()`, `isDirected()`: Standard getters for graph properties.

### `GraphVisualData.java`
- `addNode(int id, double x, double y)`: Registers a node's visual coordinates.
- `addEdge(...)`: Adds a visual edge (weighted or unweighted).
- `buildFromGraph(Graph graph, double centerX, double centerY, double radius)`: Generates a circular visual layout for an existing logical `Graph`.

### `VisualNode.java` and `VisualEdge.java`
- Data classes that store the physical `x`, `y` coordinates of a node and the `u`, `v`, `weight` of an edge for JavaFX rendering.

---

## `step` Package

### `Step.java`
- `nodeStep(StepType type, int node)`: Creates a new node-centric playback step.
- `edgeStep(StepType type, int from, int to)`: Creates a new edge-centric playback step.

### `StepType.java`
- Enum containing all possible actions during an algorithm, such as `VISIT_NODE`, `EXPLORE_EDGE`, `ADD_TO_QUEUE`, `EDGE_SELECTED`, etc.

---

## `ui` Package

### `MainApp.java`
- `start(Stage stage)`: Entry point for the JavaFX application. Sets up the canvas, legend, layout, playback buttons, and hooks up the `StepAnimator`.

### `GraphRenderer.java`
- `render(GraphVisualData data)`: Clears the canvas and draws the entire graph (nodes as Circles, edges as Lines) based on the visual data.
- `getNodeCircle(int id)`: Returns the JavaFX `Circle` for a specific node ID.
- `getEdgeLine(int u, int v)`: Returns the JavaFX `Line` for a specific edge.

### `StepAnimator.java`
- `load(List<Step> steps)`: Loads a sequence of steps generated by an algorithm.
- `play()`, `pause()`, `resume()`, `stop()`: Automatic playback controls using a JavaFX `Timeline`.
- `stepForward()`, `stepBackward()`: Manual step-by-step navigation controls.
- `processStep(Step step)`: Reads a `StepType` and updates the color/stroke of the corresponding JavaFX `Circle` or `Line`.
